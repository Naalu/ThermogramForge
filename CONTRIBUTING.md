# Contributing to ThermogramForge

First off, thank you for considering contributing to ThermogramForge! We appreciate your time and effort. Contributions are essential for making this tool better for everyone.

This document provides a high-level overview. For full details, please refer to the **[Contributor Guide](https://thermogramforge.readthedocs.io/en/latest/contributing/index.html)** in the main documentation (Link needs update when hosted).

## Ways to Contribute

* **Reporting Bugs:** If you encounter a bug, please search the [GitHub Issues](https://github.com/Naalu/ThermogramForge/issues) first to see if it has already been reported. If not, open a new issue detailing the problem, including your environment (OS, Python version, package versions) and clear steps to reproduce the bug.
* **Suggesting Enhancements:** Have an idea for a new feature or an improvement to an existing one? Open an issue to describe your suggestion, explaining the use case and potential benefits.
* **Improving Documentation:** Found typos, unclear explanations, or missing information in the documentation? Feel free to open an issue or submit a pull request with corrections.
* **Submitting Code Changes (Pull Requests):** If you'd like to fix a bug or implement a new feature, please follow the contribution workflow outlined below and detailed in the [Contribution Workflow](https://thermogramforge.readthedocs.io/en/latest/contributing/workflow.html) documentation.

## Contribution Workflow Summary

1. **Fork & Clone:** Fork the repository on GitHub and clone your fork locally.
2. **Setup:** Follow the [Getting Started](https://thermogramforge.readthedocs.io/en/latest/contributing/getting_started.html) guide to set up your development environment.
3. **Branch:** Create a new branch for your changes (e.g., `git checkout -b fix/some-bug`).
4. **Code:** Make your changes, adhering to the [Coding Conventions](https://thermogramforge.readthedocs.io/en/latest/contributing/coding_conventions.html).
5. **Check:** Run formatters and linters as described in the [Testing](https://thermogramforge.readthedocs.io/en/latest/contributing/testing.html) guide. Add tests if applicable.
6. **Commit:** Write clear, concise commit messages.
7. **Push:** Push your branch to your fork on GitHub.
8. **Pull Request:** Open a Pull Request (PR) against the `main` branch of the `Naalu/ThermogramForge` repository. Provide a clear description of your changes.
9. **Review:** Respond to any feedback from maintainers during the code review process.

*(See the full [Contribution Workflow](https://thermogramforge.readthedocs.io/en/latest/contributing/workflow.html) for details).*

## Project Structure & Conventions

* Familiarize yourself with the [Project Structure](https://thermogramforge.readthedocs.io/en/latest/contributing/project_structure.html).
* Adhere to the established [Coding Conventions](https://thermogramforge.readthedocs.io/en/latest/contributing/coding_conventions.html).

## Code of Conduct

This project and everyone participating in it are governed by the [Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code. Please report unacceptable behavior.

We look forward to your contributions!
